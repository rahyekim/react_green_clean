비동기 프로그래밍 Async

Asychronus code

        ⭐아, 이건 시간이 걸리는 비동기 작업이네? 백그라운드에 넘겨!⭐

        비동기 코드를 사용하면 프로그램이 (파일에서 데이터를 가져오는것과같은)
        시간이 오래 걸리는 작업을 시작하고
        첫 번째 작업이 완료되기 전에 다른 작업을 계속 진행할 수 있습니다.
        애플리케이션이 멈추는 것을 방지하는데, 이는 사용자 경험(UX)에 매우 중요

        일부 작업 완료하는데 시간이 걸림(네트워크 요청, 타이머, 사용자입력)
        👉 반응성을 유지하기 위해 javascript는 비동기 프로그래밍을 사용
        비동기 흐름이란 자바스크립트에서 📍특정작업이 백그라운드에서 실행📍되고,
        결과가 준비되면 처리 될 수있도록 하는 방식.

        멈춘 페이지는 깨진 페이지(하얗게 멈춤...버튼도 안눌리고 마우스스크롤도안댐)
        비동기 코드는 실행을 차단하지 않음

        자바스크립트는 서로 다른 핵심 개념을 사용하여 비동기 프로그래밍을 처리

        🔹 Synchronus : 표준 흐름이 한 줄씩 실행
        🔹 Timers : 다른 코드가 대기하는 동안 코드가 실행될 수 있도록 함 (setTimeout,setInterval)
        🔹 Callbacks : 비동기 자바스크립트를 위한 최초의 해결책 > 콜백지옥 (진동벨)
        🔹 Events : 실행 대기 중인 콜백 함수를 저장
        🔹 Promises 비동기 작업을 깔끔하게 처리하는 도구 (예약증 then()으로 알려줄게 약속)
        🔹 Async-await: 비동기 코드를 처리하는 깔끔하고 현대적인 방법 (끝판왕)

        
병렬처리 vs 비동기처리 헷갈림주의!

        📍 병렬 처리(멀티프로세스): 서로다른 process프로세스에서 여러 작업을 동시에 실행[더빠르게완료] 
                     -> 한번에 10000개의 이미지를 처리 

        🚨 비동기 처리(프로세스onlyone 싱글스레드): 동시에 실행하는 것이 아니라👉작업간 전환 []
                     -> 사용자가 스크롤하는 동안 코드실행
                    지금 이작업을 시작하세요 나는 결과를 즉시 알필요는 없어요
                    완료되면 나중에 알려주세용 
        📝요약
            📍요리사 10명, 10가지요리동시 => 병렬처리

            🚨 요리사1명, 
            고기를 불에 올려두고(시간걸림) 양파썰고있음 
            🔥작업을 번갈아 가며 빠르게 전환🔥 => 비동기처리

        
    📢 카페 주문
    1. 아메리카노 주문(비동기호출)
    2. 진동벨을 주고, 다음 손님 주문 받음
    3. 주방에서 커피(시간이걸리는작업)를 다만들면
    진동벨을 울려(콜백) 커피를 받아감    

        
        
Timers 2가지

        ✅setTimeout(함수, 시간): 지정된 시간이 지나면 딱 한번만 실행 (3초뒤에 팝업창 닫기)
        ✅setInterval(함수, 시간): 지정된 시간마다 계속 반복해서 실행 (화면에 1초마다 바뀌는시계만들기)
        
        1초= 1000ms(밀리초)
        

        🌐 웹(Web) ➡️ 3000(3초)
        사용자가 답답함을 느끼지 않는 마지노선의 시간이 보통 3초(3000ms) => 기본값 3초

        🎬 영상 ➡️ 300(0.3초)
        눈으로 인지하고 "자연스럽다, 빠르다"라고 느끼는 화면 전환 속도 300ms => 기본값 0.3초
